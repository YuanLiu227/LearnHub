# LearnHub 学习内容搜索平台 — 需求文档

## 1. 项目概述

### 1.1 项目定位
LearnHub 是一个关键词驱动的多源学习资源搜索平台，支持博主（创作者）内容自动收集和视频 URL 提交，帮助开发者从多个技术平台快速检索和发现高质量学习内容。

### 1.2 核心价值

- **多源聚合**：一次搜索覆盖 Bilibili、YouTube、编程导航、鱼皮AI导航四大数据源
- **博主追踪**：关注 Bilibili UP 主或 YouTube 频道，自动收集最新发布内容
- **视频提交**：直接粘贴 Bilibili/YouTube 视频链接，自动提取信息并入库
- **资源管理**：收藏、完成标记、归档/恢复、选择式批量删除、一键清空
- **用户配置**：每个用户独立配置 API 密钥、机场订阅链接和数据源开关
- **数据隔离**：多用户系统，每个用户数据完全隔离
- **管理后台**：独立管理界面，支持用户管理、冻结/解封、注册控制

### 1.3 目标用户

- 想要系统性学习某一技术领域的开发者
- 需要跨平台追踪特定博主/频道最新内容的用户
- 希望收藏和管理学习资源的用户
- 系统管理员（通过独立管理后台管理用户和系统设置）

## 2. 用户系统

### 2.1 角色体系

| 角色 | 说明 | 分配方式 |
|------|------|----------|
| `user` | 普通用户，可使用主前端所有功能 | 注册时默认分配 |
| `admin` | 管理员，额外可访问管理后台 | 注册邮箱与 `SMTP_USER` 匹配时自动分配 |

### 2.2 用户状态

| 状态 | 说明 |
|------|------|
| `active` | 正常状态，可正常登录和使用 |
| `frozen` | 已冻结，无法登录（返回 403） |

### 2.3 注册流程（两步验证）

```
Step 1: POST /auth/register
  提交邮箱 + 密码 + 图形验证码
    ↓
  校验图形验证码 → 检查注册开关 → 校验密码强度 → 检查邮箱是否已注册
    ↓
  发送 6 位邮箱验证码
  SMTP 已配置 → 发送邮件到注册邮箱
  SMTP 未配置 → 返回 devCode（开发模式）
    ↓
Step 2: POST /auth/register/verify
  提交邮箱 + 邮箱验证码
    ↓
  检查注册开关 → 校验邮箱验证码 → bcrypt 哈希密码 → 创建用户
    ↓
  邮箱匹配 SMTP_USER → role = admin，否则 role = user
    ↓
  返回 JWT（含 role）→ 前端存储 token → 自动登录
```

### 2.4 登录流程

| 功能 | 描述 |
|------|------|
| 登录方式 | 邮箱 + 密码 + 图形验证码，bcrypt 校验密码 |
| 状态检查 | frozen 状态返回 403 "账号已被冻结，无法登录" |
| 账号不存在 | 返回 400 "账号不存在"（避免使用 401 被 axios 拦截器跳转登录页） |
| JWT 签发 | payload 含 `{ userId, email, role }`，7 天过期 |
| Token 存储 | localStorage，通过 `Authorization: Bearer` 携带 |
| Token 恢复 | 页面刷新时读取 localStorage 的 token，调用 `GET /auth/me` 验证 |
| 401 跳转 | 所有 API 响应 401 时自动清除 token 并跳转登录页 |

### 2.5 验证码系统

#### 图形验证码
- 服务端内存存储（Map），key 为 captchaId
- 4 位数字 SVG，随机颜色、旋转、干扰线、干扰点
- 一次性使用（验证后立即删除），5 分钟 TTL（每分钟自动清理）
- 登录、注册、重发验证码均需图形验证码

#### 邮箱验证码
- 6 位数字，存储于 `verification_codes` 表（含邮箱、类型、过期时间、是否使用）
- 10 分钟有效期，验证后标记 `used = 1`
- 支持 SMTP 服务：QQ邮箱/163邮箱/Brevo/Resend
- 开发模式（SMTP 未配置）自动在控制台输出 devCode

## 3. 功能需求

### 3.1 关键词管理

| 功能 | 描述 |
|------|------|
| 添加关键词 | 用户输入关键词，系统存储并用于多源搜索 |
| 启用/暂停 | 切换关键词的搜索状态，暂停的关键词不会被搜索 |
| 归档关键词 | 软删除，不再参与搜索，已有资源保留 |
| 恢复关键词 | 将已归档关键词重新激活 |
| 永久删除 | 删除关键词及其关联的所有学习资源 |
| 关键词总览 | 展示所有关键词（含已归档），支持恢复操作 |

### 3.2 博主关注管理

| 功能 | 描述 |
|------|------|
| 添加关注 | 输入 UP 主名称（Bilibili）或频道名称（YouTube），搜索并关注 |
| 启用/暂停 | 切换博主的收集状态 |
| 归档博主 | 不再收集该博主内容，已收集资源保留 |
| 恢复博主 | 将已归档博主重新激活 |
| 永久删除 | 删除博主及其关联资源（排除收藏的资源） |
| 博主总览 | 展示所有博主（含已归档），支持恢复 |
| 收集进度 | 轮询 `/api/creators/collect/progress/:id` 实时查看进度 |
| 每日自动收集 | 每天早上 08:00 自动收集所有已启用博主的最新内容 |

### 3.3 博主内容收集策略

| 策略 | 说明 |
|------|------|
| Bilibili | 优先使用空间 API（`x/space/arc/search`），限流时回退到搜索 API（`x/web-interface/search/type`） |
| YouTube | Search API（`youtube/v3/search`）+ Video Stats API（`youtube/v3/videos`） |
| 时间筛选 | 仅收集近 7 天内容 |
| 质量阈值 | 博主收集不设播放量/点赞阈值，仅按时间筛选 |
| 进度反馈 | 异步执行，前端 500ms 轮询进度，5 分钟后自动清理进度数据 |

### 3.4 视频 URL 提交

| 功能 | 描述 |
|------|------|
| 平台选择 | Bilibili 或 YouTube，带对应平台 UI 主题色 |
| URL 解析 | Bilibili：`extractBVID()` 提取 BVID；YouTube：正则提取视频 ID |
| 信息提取 | 自动获取标题、作者、播放量、发布时间等元数据 |
| 热度计算 | `Math.min(100, Math.round(viewCount / 1000))` |
| 去重保护 | 同一 URL 不能重复提交 |
| 资源类型 | `resource_type = 'direct_video'` |

### 3.5 关键词搜索

| 功能 | 描述 |
|------|------|
| 触发方式 | 用户手动触发，在所有启用的关键词下跨源搜索 |
| 质量阈值 | Bilibili：播放≥5000、点赞≥200、收藏≥100；YouTube：播放≥10000、点赞≥500 |
| 评分排序 | 按综合评分公式计算并排序，热度倒序展示 |
| 去重机制 | Bigram Jaccard 相似度算法，标题阈值 0.75，来源阈值 0.6 |
| 进度反馈 | 异步执行，前端 500ms 轮询进度 |
| 通知面板 | 搜索完成后弹出通知，展示新增资源条目 |

### 3.6 资源管理

| 功能 | 描述 |
|------|------|
| 资源分类 | 「关键词资源」「博主资源」「视频资源」三个子标签页 |
| 综合排序 | 按发布时间降序排列（最新资源排前面） |
| 按来源筛选 | 按 Bilibili / YouTube / 编程导航 / 鱼皮AI导航 筛选 |
| 分类分页 | 三个子标签页独立从服务端按 `resource_type` 过滤分页，页码切换触发 API 请求 |
| 收藏资源 | 收藏后移出总览，仅在收藏标签页显示 |
| 取消收藏 | 检测关联实体是否存在：存在则回到总览，已删除则级联删除该资源 |
| 完成标记 | 标记学习完成（绿色边框 + 删除线样式） |
| 单条删除 | 含确认对话框 |
| 选择式批量删除 | 进入选择模式勾选多个资源，统一删除 |
| 一键清空 | 按分类（关键词/博主/视频）批量清空所有资源，含确认弹窗 |
| 知识搜索 | 全文搜索已有资源，分页浏览结果 |

| 功能 | 描述 |
|------|------|
| DeepSeek API 密钥 | 用于 AI 对话和内容真实性校验，加密存储 |
| YouTube API 密钥 | 用于 YouTube 数据源搜索和博主收集，加密存储 |

| 密钥掩码 | API 密钥响应中自动掩码（前 3 + `****` + 后 4） |

### 3.8 管理后台

独立 HTML 页面，由 Express 在 `/admin` 路由下静态提供，与主前端 SPA 完全分离。

#### 3.8.1 管理员登录

|------|------|
| 独立登录页 | 邮箱+密码+图形验证码 |
| 角色校验 | 仅 `role = admin` 可登录，非 admin 提示"该账号不是管理员" |

### 5.3 性能

- 搜索和博主收集异步执行，前端 500ms 轮询进度
- 数据库 WAL 模式提高并发读写
- ProgressEmitter 在任务完成后 5 分钟自动清理
- 博主收集和关键词搜索独立执行，互不阻塞

### 5.4 安全

- 密码 bcrypt 哈希（10 轮 salt）
- API 密钥 AES-256-GCM 加密存储
- 密钥响应自动掩码（前 3 + `****` + 后 4）
- JWT 认证中间件保护 API，用户数据 `AND user_id = ?` 隔离
- `adminRequired` 中间件保护管理 API
- 图形验证码防止机器人注册
- mihomo 配置文件和订阅 URL 存储在服务器本地
- 订阅 URL 在数据库中加密存储（AES-256-GCM）

## 6. 综合评分公式（关键词搜索）

```
综合分 = 热度(heat) × 0.4
       + 来源可信度(credibility) × 0.3
       + 时效性(timeliness) × 0.2
       + 内容类型(contentType) × 0.1
```

| 维度 | 权重 | 取值规则 |
|------|------|----------|
| 热度 | 0.4 | 基于播放量/点赞数归一化到 0-100 |
| 来源可信度 | 0.3 | 编程导航/鱼皮AI导航=0.9, YouTube=0.8, Bilibili=0.6 |
| 时效性 | 0.2 | 7 天内=1.0, 30 天内=0.7, 90 天内=0.4, 更早=0.2 |
| 内容类型 | 0.1 | AI 课程=0.9, 教程文章=0.8, 视频=0.7 |

> 博主内容收集和视频 URL 提交不经过评分系统。

### 去重算法（Bigram Jaccard）
- 标题相似度阈值：0.75
- 来源相似度阈值：0.6
- 检测到重复时保留可信度更高的条目

## 8. 设计约束

### 8.1 管理后台独立于主前端
- 主前端：React SPA，面向普通用户
- 管理后台：独立 HTML/CSS/JS 页面，由 Express 在 `/admin` 路由提供
- 两个界面完全独立，共享同一套后端 API
- 管理后台无需构建工具

### 8.2 管理员自动分配
- 注册邮箱与 `SMTP_USER` 匹配 → 自动 `role = admin`
- 无需新增独立的环境变量

### 8.3 用户配置覆盖规则
1. `user_config` 表（用户配置，API 密钥加密存储）
2. `process.env`（向后兼容）
3. 数据源开关默认 `true`
4. 其他键返回 `null`

### 8.4 代理管理
- 用户输入机场订阅链接 → 服务器自动启动 mihomo 实例
- 同一订阅 URL 共享同一 mihomo 实例（MD5 hash 前 8 位）
- 端口范围 15733~15833，自动分配和回收
- mihomo 进程崩溃自动重启（2 秒延迟）
- 服务器重启自动恢复所有代理实例
- YOUTUBE_PROXY_URL 由系统自动写入，对用户透明
