# Hot Monitor 学习内容搜索平台 — 需求文档

## 1. 项目概述

### 1.1 项目定位
Hot Monitor 是一个关键词驱动的多源学习资源搜索工具，支持博主内容自动收集，帮助用户从多个技术平台快速检索和发现高质量学习内容。

### 1.2 目标用户
- 想要系统性学习某一技术领域的开发者
- 需要跨平台追踪特定博主/频道最新内容的用户
- 希望收藏和管理学习资源的用户

### 1.3 核心价值
- **多源聚合**：一次搜索覆盖 Bilibili、YouTube、编程导航、鱼皮AI导航
- **博主追踪**：关注 UP 主/频道，自动收集最新内容（仅按时间筛选）
- **资源管理**：收藏、完成标记、归档/恢复、选择式批量删除、一键清空

## 2. 功能需求

### 2.1 关键词管理

| 功能 | 描述 |
|------|------|
| 添加关键词 | 用户输入关键词，系统存储并用于后续搜索 |
| 启用/暂停 | 切换关键词的监控状态，暂停的关键词不会被搜索 |
| 归档关键词 | 关键词归档后不再搜索，但已匹配的学习资源保留 |
| 恢复关键词 | 将已归档的关键词重新激活 |
| 永久删除 | 删除关键词及其关联的所有学习资源 |
| 关键词总览 | 展示所有关键词（含已归档），查看每个关键词的资源数，支持恢复 |

### 2.2 博主关注管理

| 功能 | 描述 |
|------|------|
| 添加关注 | 输入 UP 主名称（Bilibili）或频道 @handle（YouTube），关注博主 |
| 启用/暂停 | 切换博主的收集状态 |
| 归档博主 | 不再收集该博主内容，已收集的资源保留 |
| 恢复博主 | 将已归档的博主重新激活 |
| 永久删除 | 删除博主及其关联的所有资源 |
| 博主主页 | 点击外部链接图标跳转到博主主页 |
| 博主总览 | 展示所有博主（含已归档），支持恢复 |

### 2.3 博主内容收集

| 功能 | 描述 |
|------|------|
| 触发收集 | 用户手动触发，系统从所有已启用的博主收集最新视频 |
| 独立于关键词 | 博主收集不经过关键词搜索流程，不使用热度/质量阈值 |
| 时间筛选 | 仅按发布时间收集近期内容（默认近 7 天） |
| 进度反馈 | 实时展示收集进度 |

### 2.4 学习资源搜索（关键词）

| 功能 | 描述 |
|------|------|
| 触发搜索 | 用户手动触发，在所有启用的关键词下跨源搜索 |
| 质量阈值 | 关键词搜索的视频需满足最低播放/点赞/收藏阈值 |
| 进度反馈 | 实时展示搜索的阶段 |
| 结果存储 | 搜索结果持久化到 SQLite 数据库 |
| 综合评分 | 每条资源按公式计算综合分并排序展示 |

### 2.5 资源管理

| 功能 | 描述 |
|------|------|
| 资源列表 | 分页展示资源，分「关键词资源」和「博主资源」两个子标签 |
| 收藏资源 | 点击星标收藏，收藏后移出资源总览，仅在「收藏资源」标签页显示 |
| 取消收藏 | 取消收藏后，如关联实体仍存在则回到总览，否则级联删除 |
| 完成标记 | 标记学习完成，绿色边框 + 删除线视觉反馈 |
| 单条删除 | 含确认对话框防止误操作 |
| 选择式批量删除 | 进入选择模式勾选多个资源，统一删除 |
| 一键清空 | 清空当前标签页（关键词/博主）所有资源，含确认弹窗 |
| 知识搜索 | 全文搜索已有资源 |

### 2.6 搜索历史

| 功能 | 描述 |
|------|------|
| 搜索记录 | 展示最近的搜索记录，含时间和状态 |
| 状态提示 | 搜索中、已完成、失败等状态展示 |

### 2.7 配置管理

| 功能 | 描述 |
|------|------|
| 邮箱配置 | 设置通知接收邮箱 |

### 2.8 数据源

| 数据源 | 接口方式 | 鉴权 | 用途 |
|--------|----------|------|------|
| Bilibili | 公开搜索 API + 空间 API | 无需密钥 | 关键词搜索 + 博主关注 |
| YouTube | Data API v3 | API Key | 关键词搜索 + 博主关注 |
| 编程导航 (codefather.cn) | 文章列表 API | 无需密钥 | 关键词搜索 |
| 鱼皮AI导航 (ai.codefather.cn) | 文章列表 API | 无需密钥 | 关键词搜索 |

## 3. 非功能需求

### 3.1 技术栈
- **前端**：React 19 + Vite 6 + TypeScript + Tailwind CSS 4 + Zustand 5 + motion/react
- **后端**：Node.js + Express 5 + TypeScript (tsx 运行时)
- **数据库**：SQLite (better-sqlite3)，WAL 模式
- **AI 服务**：DeepSeek 官方 API (DeepSeek V4 Flash)，用于内容摘要和质量评估

### 3.2 性能
- 搜索请求异步执行，前端实时轮询进度
- 数据库启用 WAL 模式提高并发读写性能
- 博主收集和关键词搜索独立执行，互不阻塞

### 3.3 安全
- API 密钥通过环境变量注入，不硬编码
- 前端通过 Vite proxy 转发 API 请求，避免跨域问题
- .env 文件已被 .gitignore 排除，防止密钥泄露

## 4. 综合评分公式（关键词搜索）

```
综合分 = 热度(heat) × 0.4
       + 来源可信度(credibility) × 0.3
       + 时效性(timeliness) × 0.2
       + 内容类型(contentType) × 0.1
```

| 维度 | 权重 | 取值规则 |
|------|------|----------|
| 热度 | 0.4 | min(100, 播放量/1000)，归一化到 0~1 |
| 来源可信度 | 0.3 | 编程导航/鱼皮AI导航=0.9, Bilibili=0.6, YouTube=0.8 |
| 时效性 | 0.2 | 7天内=1.0, 30天内=0.7, 90天内=0.4, 更早=0.2 |
| 内容类型 | 0.1 | AI课程=0.9, 教程文章=0.8, 视频=0.7 |

> 注：博主内容收集不经过评分系统，仅按发布时间排序。

## 5. API 接口一览

### 5.1 关键词
| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/keywords` | 获取活跃关键词列表 |
| GET | `/api/keywords/all` | 获取全部关键词（含归档） |
| POST | `/api/keywords` | 添加关键词 |

| POST | `/api/keywords/archive` | 归档关键词 |
| POST | `/api/keywords/restore` | 恢复关键词 |
| DELETE | `/api/keywords/permanent` | 永久删除关键词 |

| PATCH | `/api/creators/:id` | 更新博主 |
| POST | `/api/creators/archive/:id` | 归档博主 |
