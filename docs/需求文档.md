# LearnHub 学习内容搜索平台 — 需求文档

## 1. 项目概述

### 1.1 项目定位
LearnHub 是一个关键词驱动的学习内容搜索工具，帮助用户从多个技术平台快速检索和发现高质量学习资源。

### 1.2 目标用户
- 想要系统性学习某一技术领域的开发者
- 需要快速获取特定技术关键词相关教程/视频/文章的学习者
- 希望跨平台聚合学习资源的用户

### 1.3 核心价值
- **多源聚合**：一次搜索覆盖 Bilibili、YouTube、编程导航、鱼皮AI导航等多个技术平台
- **质量评估**：综合评分体系 (热度 + 来源可信度 + 时效性 + 内容类型) 帮助筛选优质内容
- **关键词管理**：灵活的关键词增删改查，支持归档 (保留资源) 和永久删除

## 2. 功能需求

### 2.1 关键词管理

| 功能 | 描述 |
|------|------|
| 添加关键词 | 用户输入关键词，系统存储并用于后续搜索 |
| 启用/暂停 | 切换关键词的监控状态，暂停的关键词不会被搜索 |
| 归档关键词 | 关键词归档后不再搜索，但已匹配的学习资源保留 |
| 永久删除 | 删除关键词及其关联的所有学习资源 |
| 关键词总览 | 展示所有关键词 (含已归档)，查看每个关键词的资源数 |

### 2.2 学习资源搜索

| 功能 | 描述 |
|------|------|
| 触发搜索 | 用户手动触发，系统在所有启用的关键词下跨源搜索 |
| 进度反馈 | 实时展示搜索的阶段 (started → collecting → matching → verifying → completed) |
| 结果存储 | 搜索结果持久化到 SQLite 数据库 |
| 综合评分 | 每条资源按公式计算综合分并排序展示 |

### 2.3 资源管理

| 功能 | 描述 |
|------|------|
| 资源列表 | 分页展示所有已匹配的学习资源，按综合评分降序排列 |
| 单条删除 | 用户可删除单条学习资源 |
| 新资源标记 | 最近一次搜索新增的资源带有标记 |
| 知识搜索 | 全文搜索已有资源 (标题、摘要、关键词) |

### 2.4 配置管理

| 功能 | 描述 |
|------|------|
| 通知开关 | 启用/禁用浏览器通知和邮件通知 |
| 邮箱配置 | 设置通知接收邮箱 |

### 2.5 数据源

| 数据源 | 接口方式 | 搜索策略 |
|--------|----------|----------|
| Bilibili | 公开搜索 API | `order=click` 按播放量排序，过滤低质内容 |
| YouTube | YouTube Data API | 按关键词搜索视频，过滤低播放量内容 |
| 编程导航 (codefather.cn) | 文章列表 API | 搜索技术文章/教程 |
| 鱼皮AI导航 (ai.codefather.cn) | 文章列表 API | 搜索 AI 课程/教程 |

## 3. 非功能需求

### 3.1 技术栈
- **前端**：React 19 + Vite 6 + TypeScript + Tailwind CSS 4 + Zustand
- **后端**：Node.js + Express 5 + TypeScript (tsx 运行时)
- **数据库**：SQLite (better-sqlite3)
- **AI 服务**：SiliconFlow API (DeepSeek-V3)，用于内容总结和质量评估

### 3.2 性能
- 搜索请求异步执行，前端实时轮询进度
- 数据库启用 WAL 模式提高并发读写性能
- 单次搜索超时由后端控制，不阻塞用户操作

### 3.3 安全
- API 密钥通过环境变量注入，不硬编码
- 前端通过 Vite proxy 转发 API 请求，避免跨域问题
- 支持 CORS 配置

## 4. 综合评分公式

```
综合分 = 热度(heat) × 0.4
       + 来源可信度(credibility) × 0.3
       + 时效性(timeliness) × 0.2
       + 内容类型(contentType) × 0.1
```

| 维度 | 权重 | 取值规则 |
|------|------|----------|
| 热度 | 0.4 | min(100, 播放量/1000)，归一化到 0~1 |
| 来源可信度 | 0.3 | 编程导航/鱼皮AI导航=0.9, YouTube=0.7, Bilibili=0.6 |
| 时效性 | 0.2 | 7天内=1.0, 30天内=0.7, 90天内=0.4, 更早=0.2 |
| 内容类型 | 0.1 | AI课程=0.9, 教程文章=0.8, 视频=0.7 |

## 5. API 接口一览

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/keywords` | 获取活跃关键词列表 |
| GET | `/api/keywords/all` | 获取全部关键词（含归档） |
| POST | `/api/keywords` | 添加关键词 |
| POST | `/api/keywords/archive` | 归档关键词 |
| DELETE | `/api/keywords/permanent` | 永久删除关键词 |
| PATCH | `/api/keywords` | 更新关键词 |
| POST | `/api/monitor` | 触发全量搜索 |
| GET | `/api/monitor/progress/:id` | 查询搜索进度 |
| GET | `/api/monitor/history` | 获取搜索历史 |
| GET | `/api/dashboard/stats` | 获取统计概览 |
| GET | `/api/dashboard/hotspots` | 获取资源列表（分页） |
| GET | `/api/dashboard/search` | 搜索已有资源 |
| DELETE | `/api/dashboard/resource/:id` | 删除单条资源 |
| GET | `/api/config` | 获取配置 |
| PUT | `/api/config` | 更新配置 |
| GET | `/api/health` | 健康检查 |
