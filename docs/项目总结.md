# LearnHub 学习内容搜索平台 — 项目总结

## 项目背景

本项目最初定位为"AI 热点监控系统" (Hot Monitor)，用于自动采集 AI 领域热点资讯。在开发过程中，根据实际使用需求重新定位为**关键词驱动的学习内容搜索工具 (LearnHub)**，帮助用户通过关键词从多个技术平台快速获取高质量学习资源。

## 数据流

```
用户添加关键词 → 触发搜索
                    ↓
          ┌─── Bilibili API (公开搜索)
          ├─── YouTube Data API
          ├─── 编程导航 API
          └─── 鱼皮AI导航 API
                    ↓
              综合评分计算
                    ↓
              存入 SQLite
                    ↓
          ┌─── 前端展示资源列表
          ├─── 浏览器/邮件通知
          └─── 知识搜索（全文检索）
```

## 数据源对比

| 数据源 | 鉴权方式 | 内容特点 | 评分权重 |
|--------|----------|----------|----------|
| Bilibili | 无需密钥 | 视频教程，覆盖面广，质量参差 | 可信度 0.6 |
| YouTube | API Key | 高质量英文教程，内容丰富 | 可信度 0.7 |
| 编程导航 | 无鉴权 | 技术文章/教程，质量高 | 可信度 0.9 |
| 鱼皮AI导航 | 无鉴权 | AI 课程/教程，质量高 | 可信度 0.9 |

## 关键设计决策

### 1. 综合评分系统
从单一热度排序改为多维度评分：热度(40%) + 可信度(30%) + 时效性(20%) + 内容类型(10%)，更符合学习场景。

### 2. 两层关键词管理
- **归档**：软删除，保留学习资源（适用于暂时不需要但仍想保留资源的场景）
- **永久删除**：硬删除关键词及所有关联资源（适用于关键词废弃的场景）

### 3. 异步搜索 + 轮询进度
搜索操作完全异步，前端通过轮询 `/api/monitor/progress/:id` 获取实时进度，避免请求超时。

### 4. AI 服务命名修正
实际调用的是 SiliconFlow API (DeepSeek-V3 模型)，将文件名从 `openrouter.ts` 修正为 `siliconflow.ts`。

## 工程问题修复

项目经过代码审查后完成了以下修复：

| 问题 | 修复方式 |
|------|----------|
| 3 个死服务文件 (firecrawl/twitter) | 删除 |
| `verifiedCount` 永不递增 | 在 DB insert 成功后自增 |
| 循环内动态 import | 改为文件顶部静态 import |
| Nodemailer 模块级创建 | 改为惰性初始化 |
| `as any` 滥用 (30+ 处) | 替换为具体类型接口 |
| ProgressEmitter 内存泄漏 | 完成/错误后定时清理 |
| 数据库路径硬编码 | 支持 `DB_PATH` 环境变量覆盖 |
| 前端 3 个死组件 | 删除 |
| 接口重复声明 | 删除冗余声明 |
| 文件名误导 | `openrouter.ts` → `siliconflow.ts` |

## 开发环境

- Node.js >= 18
- npm workspaces (client/ + server/)
- 前端端口: 5173
- 后端端口: 3001
- Vite proxy 将 `/api` 转发到后端

## 部署要点

1. 配置环境变量 `SILICONFLOW_API_KEY`、`YOUTUBE_API_KEY` 等
2. 数据库文件默认生成在 `server/data/hot-monitor.db`
3. 可通过 `DB_PATH` 环境变量自定义数据库路径
4. 前端构建后通过 `dist/` 目录部署
